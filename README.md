# COSS 성과지표 총괄관리 시스템

강원대학교 **데이터보안·활용 혁신융합대학사업단(COSS)**의 성과지표를 총괄 관리하기 위한
내부 업무용 웹 대시보드입니다. 2026년도 3차년도 성과지표의 목표값·실적값·달성률·대학별
배부값을 한눈에 확인하고, 주관대학(강원대학교) 및 참여대학의 실적을 취합·검증·관리하는
데 사용합니다.

> 참고: 이 프로젝트는 참고 사이트(cossmanage.netlify.app)의 화면 구성과 분위기를 참고하여
> **처음부터 새로 제작**한 독립 프로젝트입니다. 기존 사이트의 코드를 복사하거나 수정하지 않았습니다.

## 주요 특징

- 로그인 후 **관리자(admin) / 대학 담당자(university)** 권한에 따라 메뉴와 데이터 접근 범위가 달라집니다.
- **사업단이 실제로 운영 중인 구글시트(대학별 탭 구조)를 그대로 데이터베이스로 사용**합니다.
  별도 스키마로 시트를 새로 만들 필요 없이, 기존 `총괄`/대학별 탭(강원대학교, 아주대학교,
  충남대학교, 한양대학교ERICA, 영남이공대학교)에 입력된 지표·목표값·실적값을 Google Apps
  Script가 직접 읽고 씁니다.
- **구글시트 API 연동 URL이 설정되지 않았거나 호출에 실패하면 자동으로 샘플 데이터로 대체(fallback)**되어
  화면이 비어 보이지 않습니다.
- 대시보드, 성과지표 총괄 현황, 지표 대분류별 상세, 대학별 배부·달성 관리, 우선 관리 지표,
  목표값 설정, 사용자 관리, 수정 이력 등 9개 화면을 제공합니다.
- **조회 전용 화면(대시보드/총괄 현황/지표 대분류별 상세/우선 관리 지표/수정 이력)은 30초마다
  자동으로 최신 구글시트 데이터를 다시 불러옵니다.** 화면을 새로고침하지 않아도 구글시트를
  수정하면 잠시 후 자동으로 반영됩니다(`src/utils/useAutoRefresh.ts`). 입력 화면(대학별
  배부·달성 관리, 목표값 설정, 사용자 관리)은 작성 중인 내용이 덮어써지지 않도록 자동
  새로고침 대상에서 제외했습니다.

## 구글시트 구조 (실제 운영 시트 기준)

이 프로젝트는 **처음부터 새로 만든 시트 스키마가 아니라, 사업단이 이미 쓰고 있는 시트를
그대로** 사용하도록 만들어졌습니다.

### 대학별 탭 (강원대학교 / 아주대학교 / 충남대학교 / 한양대학교ERICA / 영남이공대학교)

5개 탭 모두 동일한 지표 목록·순서를 공유하며, 각 탭은 그 대학의 배부 목표값과 실적을 담습니다.
1~3행은 제목/헤더이고, **4행부터 실제 데이터**가 시작됩니다.

| 열 | 내용 | 비고 |
|---|---|---|
| A | 지표 대분류 | 병합 셀 (예: 교원확보지수, 교육과정 및 교과목 운영지수 등) |
| B | 지표 중분류 | 병합 셀 |
| C | 증빙 링크 | 읽지 않음 (증빙 제출 여부는 이 시트에서 관리하지 않음) |
| D | 지표명 | 5개 탭에서 동일해야 대학 간 매칭이 됩니다 |
| E | 담당자 | 병합 셀일 수 있음 |
| F | 3차년도 목표값(배부 목표값) | **읽기·쓰기 모두 지원** |
| G | 3차년도 실적값 | **읽기·쓰기 모두 지원** — 대학 담당자가 이 화면에서 입력하면 이 셀에 반영됩니다 |
| H | 목표값-실적(차이) | 참고용, 대시보드는 달성률을 직접 계산합니다 |
| I | 비고/내용 | 탭마다 있을 수도 없을 수도 있어 **읽기만** 지원 (쓰기 미지원) |

> 대학별 탭에 지표를 추가/삭제하거나 순서를 바꾸면, 강원대학교 탭을 기준으로 지표 목록이
> 다시 만들어지므로 **5개 탭의 지표명과 순서를 동일하게 유지**해 주세요.

### `users` / `logs` 탭 (계정·이력 관리용, 최초 실행 시 자동 생성)

기존 운영 시트에는 로그인 계정이나 수정 이력을 관리하는 탭이 없으므로, Apps Script가
최초 호출될 때 자동으로 아래 2개 탭을 만들고 기본 계정을 채워 넣습니다. 기존 탭은 전혀
건드리지 않습니다.

**`users`**: `user_id, name, email, role, university_name, password, status`
**`logs`**: `log_id, timestamp, user_id, user_name, university_name, action, sheet_name, row_id, field_name, old_value, new_value`

## 데이터 처리 규칙

- 달성률 = `실적(G열) / 목표값(F열) * 100` (소수 첫째 자리 반올림), 전체 달성률은 5개 대학의
  목표값·실적값을 합산해 계산합니다.
- 목표값이 0이거나 비어 있으면(`-`) 달성률은 `N/A` 처리
- 달성률 100 이상: **정상** / 80 이상 100 미만: **주의** / 80 미만: **미달**
- 실적값이 전혀 입력되지 않은 지표: **미제출**
- 목표값·실적값에는 음수를 입력할 수 없음 (프론트엔드 + Apps Script 양쪽에서 검증)
- 저장 성공/실패는 화면 우측 하단 토스트 알림으로 표시
- **증빙 제출 여부**는 실제 시트에서 관리하지 않아 항상 "해당없음"으로 표시되며, 대학별
  관리 화면에서 증빙/비고를 수정해도 실시간 연동 중에는 저장되지 않습니다(화면에 안내 문구 표시).
- **전체 목표값**은 대학별 배부 목표값(F열)의 합계로 자동 계산되는 값이라 직접 수정할 수
  없고, 대학별 배부 목표값만 수정할 수 있습니다.

## Google Apps Script API

`google-apps-script/Code.gs` 파일 하나로 구성되어 있으며, 아래 액션을 `?action=` 파라미터
(GET) 또는 JSON body의 `action` 필드(POST)로 라우팅합니다.

| action | 메서드 | 설명 |
|---|---|---|
| `getDashboardData` | GET | 대시보드 요약 통계 (대분류별 평균 달성률 포함) |
| `getIndicators` | GET | 지표 목록 (category 파라미터로 대분류 필터) |
| `getUniversityResults` | GET | 대학별 실적 (role, university_name으로 범위 제한) |
| `updateUniversityResult` | POST | 실적값(G열)/배부 목표값(F열) 수정 + 달성률 자동계산 + 이력 저장 |
| `updateTarget` | POST | 항상 실패 응답 반환 (전체 목표값은 대학별 배부 목표값의 합계이므로 직접 수정 불가) |
| `getPriorityIndicators` | GET | 우선 관리 지표 자동 추출 |
| `login` | POST | 시트 기반 간이 로그인 |
| `getUsers` / `upsertUser` | GET / POST | 사용자 관리 화면 지원 |
| `getChangeLogs` | GET | 수정 이력 화면 지원 |

### Apps Script 배포 방법

1. 성과지표 구글시트에서 `확장 프로그램 > Apps Script`를 엽니다.
2. 기본 `Code.gs` 내용을 지우고 이 저장소의 `google-apps-script/Code.gs` 전체 내용을 붙여넣고 저장합니다.
3. 대학별 탭 이름이 `강원대학교`, `아주대학교`, `충남대학교`, `한양대학교ERICA`, `영남이공대학교`와
   정확히 일치하는지 확인합니다 (한양대학교ERICA는 띄어쓰기 없음에 주의).
4. `배포 > 새 배포 > 유형: 웹 앱`을 선택합니다.
   - 실행 계정: **나(본인)**
   - 액세스 권한: **전체 공개(익명 사용자 포함)**
5. **배포**를 누르고 권한 승인 요청이 뜨면 본인 계정으로 승인합니다.
6. 배포 후 발급되는 웹 앱 URL(`https://script.google.com/macros/s/.../exec`)을 복사합니다.
7. 프론트엔드 `.env` 파일의 `VITE_GAS_API_URL`에 이 URL을 설정합니다.
8. 배포 후 아무 조회 화면이나 한 번 열면 `users`/`logs` 탭이 자동 생성됩니다.

## 사용 방법

### 관리자(admin)

1. 로그인 후 전체 메뉴에 접근할 수 있습니다.
2. **대시보드**에서 전체 달성률, 지표 대분류별 평균 달성률, 대학별 비교, 우선 관리 지표를 확인합니다.
3. **성과지표 총괄 현황**에서 지표구분(대분류)·대학명·달성상태로 필터링하고 엑셀(CSV)로 내려받습니다.
4. **대학별 배부·달성 관리**에서 모든 대학의 실적을 조회·수정할 수 있습니다 (실적값은 시트에 바로 반영).
5. **목표값 설정**에서 대학별 배부 목표값을 수정합니다(수정일·수정자 자동 기록).
6. **사용자 관리**에서 계정을 추가하고 권한·소속대학·사용 여부를 설정합니다.
7. **수정 이력**에서 모든 변경 내역을 추적합니다.

### 대학 담당자(university)

1. 로그인 후 목표값 설정, 사용자 관리, 수정 이력 메뉴는 표시되지 않습니다.
2. **대학별 배부·달성 관리**에서 본인 소속 대학의 실적만 조회·입력할 수 있습니다.
3. 실적값을 입력하고 **저장**하면 구글시트의 해당 대학 탭 G열에 바로 반영되고, 달성률이 자동 계산됩니다.
4. 저장 성공/실패 알림이 화면 우측 하단에 표시됩니다.
